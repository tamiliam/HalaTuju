# Retrospective — STPM Headlines Sprint (2026-03-15)

## What Was Built

Added quirky BM headlines (emoji + tagline) to all 951 non-UiTM STPM courses, displayed as subtitles on the course detail page via a shared `CourseHeader` component.

1. **Shared CourseHeader component**: Extracted Type badge + Level badge + title + subtitle into a reusable component used by both SPM and STPM detail pages. Badge maps (`TYPE_LABELS`, `TYPE_COLORS`, `LEVEL_COLORS`) moved from CourseCard to shared `courseBadges.ts`.

2. **STPM headline column**: Added `headline` CharField(max_length=200) to StpmCourse model, API returns it, frontend shows `headline || university` as subtitle.

3. **951 BM headlines generated**: First ~250 handcrafted via direct SQL batches, remaining ~701 via bulk SQL with category-based emoji mapping and regex extraction from course names. 64 problematic headlines (Kepujian extraction, university names) fixed individually.

4. **Management command**: `generate_stpm_headlines.py` for future use with Gemini API (updated to use new `google.genai` Client API, not deprecated `google.generativeai`).

## What Went Well

- **Shared CourseHeader** eliminated duplicated header markup across SPM and STPM detail pages
- **Bulk SQL generation** was much faster than Gemini API calls for 951 courses — completed in minutes vs hours
- **Category-based emoji mapping** produced consistent, relevant headlines for most courses
- **Fallback strategy** (`headline || university`) means the feature degrades gracefully for any missing data

## What Went Wrong

1. **Regex extraction picked up "Kepujian" from parentheses (60 courses)**
   - *Symptom*: Headlines showed "Kepujian: ..." instead of the course subject
   - *Root cause*: The regex `\(([^)]+)\)` extracted the last parenthesised text, which for many courses was "(Kepujian)" — the honours designation, not the specialisation
   - *Fix*: Manually crafted headlines for all 60 courses. Future bulk generation should exclude parenthetical text matching known non-subject patterns (Kepujian, Mod 3u1i, campus names)

2. **Regex extracted university campus names (3 courses)**
   - *Symptom*: Headlines showed "Universiti Putra Malaysia Sarawak: ..."
   - *Root cause*: Same regex — campus name was in parentheses at the end of course name
   - *Fix*: Manually replaced. Same prevention as above.

3. **Initial headline tone was too colloquial**
   - *Symptom*: User said "Could you make it a little less colloquial"
   - *Root cause*: First batch used slang-like BM ("Bentuk Atlet Masa Depan") targeting too young an audience
   - *Fix*: Revised to polished but engaging style ("Latih Atlet Bertaraf Dunia"). Documented tone guideline: polished, student-friendly, not slang.

## Design Decisions

- **Course content in BM in DB, not i18n**: Headlines stored as a database column, not i18n message keys. BM is canonical. EN/TA translations may come later as additional DB columns. i18n system is for static UI strings only. (See decisions.md for full rationale.)
- **Claude-generated over Gemini-generated**: For this sprint, headlines were generated by Claude via Supabase SQL rather than the Gemini management command. Faster, no API key needed locally, and quality could be reviewed in real-time.
